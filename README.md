# Paint App (Qt/C++)
Проект представляет собой рисовалку, разработанный на C++ с использованием фреймворка Qt. Основная цель проекта — глубокое изучение и практическое применение ключевых принципов ООП, паттернов проектирования, а также создание модульной, расширяемой архитектуры через динамические библиотеки (плагины).

## Ключевые особенности и архитектура
### 1. Модульность и расширяемость (Плагины)
Каждая фигура/объект для рисования является изолированным модулем (классом). Программа поддерживает динамическую загрузку внешних модулей в формате DLL.
* **Основной проект предоставляет абстрактные интерфейсы.**
* **Сторонние модули компилируются отдельно и динамически подгружаются ядром приложения при запуске и при нажатии кнопки на интерфейсе.**
* **Репозиторий с шаблоном для создания собственных модулей: [OOP_paint_moduls](https://github.com/watari1337/OOP_paint_moduls).**

### 2. Паттерны проектирования
Фабричный метод (Factory Method) + Мейерс Синглтон (Singleton): Фабрика объектов реализована как синглтон. Внедрен механизм саморегистрации классов в фабрике. Новые модули регистрируют себя автоматически при загрузке DLL, что исключает необходимость изменять код самого ядра (соблюдение принципа Open/Closed).

### 3. Сложная иерархия наследования
Для демонстрации возможностей полиморфизма и наследования, помимо статичных фигур, реализован специальный класс анимированного объекта:

* Объект постоянно движется вперед по круговой траектории.
* Реализована логика "телепортации" границ, благодаря чему объект никогда не покидает пределы экрана и циклично появляется с противоположной стороны через определенный промежуток времени.

### 4. Сериализация и десериализация (Сохранение проектов)
Поддерживается сохранение и загрузка состояния холста двумя независимыми способами:
* **Бинарный формат**
    Для оптимизированного по размеру и скорости сохранения.
* **Текстовый формат**
    Для удобочитаемости и возможности ручного редактирования структуры сцены.

### 2. Подключение абстрактных классов:
В файле конфигурации проекта (.pro или CMakeLists.txt в зависимости от вашей сборки) укажите корректный путь к заголовочным файлам базовых модулей из основного проекта.
**Но я советую через .pri который уже настроен в основном проекте**

## Пример для .pro файла: укажите путь к абстрактным модулям основного проекта
INCLUDEPATH += "путь/к/основному/проекту/core/abstract"

Настройка путей вывода (Output Paths):
По умолчанию в проекте настроен автоматический экспорт скомпилированных .dll файлов прямо в рабочую директорию основного приложения. Перед сборкой убедитесь, что вы адаптировали эти пути под свою файловую систему:

## Настройте путь, куда основное приложение складывает плагины (например, папка /plugins)
DESTDIR = "путь/к/основному/проекту/bin/plugins"

Сборка: Откройте проект модулей в Qt Creator, настройте пути, как указано выше, и запустите сборку (Build). Сгенерированная DLL автоматически попадет в папку плагинов основного приложения и будет подгружена при следующем старте.
